# ST Market Intelligence

## 1. هدف المشروع

بناء منصة مستقلة بالكامل لتحليل الأسهم والخيارات الأمريكية، تعتمد على بيانات Massive وFinnhub فقط، وتحول البيانات المعقدة إلى نتائج واضحة وسهلة الفهم لجميع المتداولين.

## 2. مبادئ المشروع

- المشروع مستقل 100% عن أي مشروع أو بوت قديم.
- لا يتم نسخ أي كود من المشاريع السابقة.
- استخدام Massive API لبيانات الخيارات.
- استخدام Finnhub API لسعر السهم.
- كل معلومة يجب أن تساعد المستخدم في اتخاذ قرار.
- الواجهة يجب أن تكون سهلة للمبتدئ وعميقة للمحترف.
- لا يتم إصدار توصية قبل التحقق من جودة وحداثة البيانات.
- جميع المحركات قابلة للاختبار بشكل مستقل.

## 4. مكونات النظام

### Data Engine

مسؤول عن:

- جلب أسعار الأسهم.
- جلب عقود الخيارات.
- تنظيف البيانات.
- توحيد صيغة البيانات.
- التحقق من حداثة البيانات.
- حفظ Snapshots.
- تسجيل الأخطاء واستهلاك API.

### Analysis Engine

يحتوي على:

- Gamma Module
- Options Flow Module
- Momentum Module
- Market Context Module
- Contract Quality Module
- Scoring Module

### API

مسؤول عن تقديم النتائج للموقع بشكل آمن وموحد.

### Website

مسؤول عن:

- لوحة السوق.
- صفحة تحليل السهم.
- الشارتات.
- المفضلة.
- التنبيهات.
- الحسابات والاشتراكات لاحقًا.

## 5. المخرجات الأساسية لكل سهم

- السعر الحالي.
- نسبة التغير.
- الاتجاه.
- حالة الفرصة.
- درجة من 100.
- درجة الثقة.
- Gamma Flip.
- Call Wall.
- Put Wall.
- Net Gamma.
- Net Delta.
- سيطرة CALL وPUT.
- العقود غير الاعتيادية.
- أفضل عقد.
- منطقة الدخول.
- مستوى إلغاء السيناريو.
- الأهداف.
- تفسير عربي واضح.

## 6. حالات الإشارة

- NEUTRAL — محايد
- WATCH — مراقبة
- READY — جاهزة
- ACTIVE — مفعلة
- WEAKENING — تضعف
- INVALIDATED — ملغاة

1. توثيق المشروع.

5. حفظ Snapshots لعشرة رموز.
6. بناء Gamma Module.
7. بناء Flow Module.
8. بناء Momentum Module.
9. بناء Scoring Module.
10. بناء API.

## 8. معيار نجاح المرحلة الأولى

تعتبر مرحلة Data Engine ناجحة عند تحقق الآتي:

- جلب الرموز العشرة بدون فشل.
- التحقق من حداثة البيانات.
- التعامل مع القيم الناقصة.
- عدم تكرار اللقطات.
- تسجيل أخطاء API.
- تسجيل عدد الطلبات.
- حفظ وقت كل Snapshot.
- عمل المحرك بشكل مستقل عن الموقع.
